## 2. Components & Behavior

### 2.1 `src/components/ui/WheelPicker.tsx` (new)

Generic snap-scroll column:

- Props: `options: { value: number; label: string }[]`, `value: number`, `onChange: (v: number) => void`, `ariaLabel: string`.
- Fixed item height 40px, container shows 5 rows (200px) with 80px top/bottom spacers so first/last items can center; CSS `snap-y snap-mandatory`, items `snap-center`.
- Selection commits on scroll settle: `scrollend` event where supported, else a 150ms scroll-idle debounce. Selected index = `round(scrollTop / 40)`, clamped to bounds.
- Tapping an item selects it (smooth-scrolls to center).
- Controlled: when `value` changes externally (e.g., sibling auto-push), the wheel scrolls to the new position without emitting `onChange` (guard flag against feedback loops).
- Center row is visually emphasized (bold ink, `bg-raised` band); fade masks top/bottom.
- A11y: container `role="listbox"` + `aria-label`; items are `<button role="option" aria-selected>`.

## 3. Error handling summary

- Wheel: clamped index on settle; sibling auto-push via `clampTimeRange`; hint on adjustment; pre-submit + server validation backstops.
- Zoom/toggle: localStorage access wrapped in try/catch (private mode); unknown stored zoom value falls back to `md`.
- Event-days filter: if filtering yields zero pages (defensive; cannot happen for a valid event), the toggle is ignored and the full view renders.
- WheelPicker with an empty `options` array renders nothing (guard, not crash).
