## 2. Auth & Session (A)

### 2.1 Instant sign-in state

Root cause: `linkWithPopup` upgrades the SAME Firebase User object; `onAuthStateChanged` does not re-fire; the store's reference is unchanged â†’ no re-render. Fix in `authStore`:

- New state `isGoogleUser: boolean` (derived: `user != null && !user.isAnonymous`).
- Set inside the `onAuthStateChanged` listener; set to `true` explicitly at the end of a successful `signInWithGoogle()` (from `auth.currentUser`); set to `false` at the start of `signOut()`.
- `AppHeader` and `DashboardPage` switch from `user.isAnonymous` checks to `isGoogleUser`.

### 2.2 Sign out

`AppHeader` signed-in cluster: `Dashboard` link + `Sign out` ghost button (calls `authStore.signOut()`; the existing listener re-signs anonymously, so voting identity continues). Header flips instantly via `isGoogleUser`.

### 2.3 Auto-logoff (1h inactivity)

New hook `src/hooks/useAutoLogoff.ts`, mounted once in `App`:

- Records activity on `pointerdown` and `keydown` (throttled to one localStorage write per 30s) at key `s2g-last-activity` (epoch ms) â€” localStorage so all tabs share one activity clock.
- A 60s interval plus a mount-time check: if `isGoogleUser` and `now - lastActivity > 60 * 60_000` â†’ `signOut()`.
- Guards: localStorage try/catch (private mode â†’ hook inert); missing key initializes to now; anonymous users never auto-signed-out.

## 6. Join flow with claim + passcode (E)

### 6.1 Data

- Participant doc gains public `protected?: boolean` (UI lock badge only).
- New client-inaccessible subcollection `events/{slug}/secrets/{participantId}`: `{ passcodeHash: string, salt: string }` (scrypt via node `crypto.scryptSync`, N=16384; hex). Rules: `allow read, write: if false;`.

### 6.2 `joinWithName` callable (`functions/src/joinWithName.ts`, region us-central1)

Input: `{ slug: string, name?: string, passcode?: string, claimParticipantId?: string }` â€” exactly one of `name` (create) or `claimParticipantId` (claim). Auth required. Flow:

1. Load event; reject missing (`not-found`) or finalized (`failed-precondition`, "Voting is closed").
2. **Create** (`name`): validate 1â€“79 chars; normalize (lowercase, trim, collapse spaces â€” matching `src/lib/nameNormalize.ts`); if any existing participant's normalized name matches â†’ `already-exists` ("That name is taken â€” claim it instead"). Create participant `{ name, uid, availability: '', protected: passcode != null, lastUpdated }` with a random id (admin `doc()` auto-id); if passcode (validated `/^[a-zA-Z0-9]{4,12}$/`, else `invalid-argument`): store scrypt hash + random salt in `secrets/{participantId}`.
3. **Claim** (`claimParticipantId`): load participant â†’ `not-found` if missing. Load its secrets doc: if exists â†’ require passcode and scrypt-verify (`timingSafeEqual`) â†’ wrong/missing = `permission-denied` ("Wrong passcode"); if no secrets doc â†’ claim freely. On success: update participant `uid` to caller's uid.
4. Return `{ participantId, name }`.

Pure decision helper `decideJoin(...)` in `functions/src/lib/joinLogic.ts` (unit-tested; hashing helpers `hashPasscode`/`verifyPasscode` alongside, also unit-tested).

## 7. Error handling summary

Auth: popup-close swallowed; auto-logoff inert without localStorage. Join: typed HttpsError codes mapped to friendly inline messages (`already-exists` / `permission-denied` / `failed-precondition` / `not-found` / `invalid-argument`); callable network failure â†’ generic inline retry message. Rules deny client secret access and direct participant creation; scrypt with per-participant salt + timingSafeEqual.
