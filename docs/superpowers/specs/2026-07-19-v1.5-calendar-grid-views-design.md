# schedule2gather v1.5 — Calendar Selection + Grid Views — Design Spec

**Date:** 2026-07-19
**Status:** Approved by user. Client-only; lands on `redesign` (PR #5).

## 1. Scope

1. Selected-dates list + count move to their OWN card below the calendar card.
2. Create-calendar row/column selection: clickable weekday headers (column) and week numbers (row), toggle semantics.
3. Desktop grid gains Week | Month | All views (All default on desktop; mobile keeps Week | Month). "Event days only" available on desktop in week/month views.
4. Mobile paint-mode toggle shown in BOTH week and month views (week no longer always-paint).
5. Desktop: no paint toggle (always paint); hovering the grid converts vertical wheel to horizontal scroll when the table overflows.

## 2. Create form (`CreateEventForm.tsx`)

### 2.1 Separate selected-dates section

Calendar `Card` keeps: mode toggle, quick-select rows (WITHOUT the count/clear cluster), calendar, range hint. NEW `Card` directly below it, rendered only when `selectedDates.length > 0`:
header row `SELECTED DATES · {n}` (micro-label style) + right-aligned `Clear all` (clears dates + range draft); body = the existing sorted removable list (`max-h-40 overflow-y-auto`, hairline dividers).

### 2.2 Weekday-column and week-row selection

- Toggle helper (pure, in `src/lib/dateRange.ts`): `toggleDays(existing: Date[], candidates: Date[]): Date[]` — if every candidate (by day) is already in `existing`, remove them all; otherwise add the missing ones. Candidates are always pre-filtered to `>= today`.
- **Weekday headers:** day-picker v9 `components.Weekday` override renders each header cell as a button; clicking weekday `w` of displayed month `m` builds candidates = all days of `m` with `getDay() === w` and `>= today`, then `toggleDays`. Desktop shows two months: each month's header operates on its own month (the override receives the rendered month context; if v9's Weekday props don't expose the month, fall back to a column-aligned clickable weekday chip-row rendered above each month grid — same handler).
- **Week rows:** `showWeekNumber` + `onWeekNumberClick(week)` — candidates = that week's days within the displayed month(s) and `>= today`, then `toggleDays`. Week-number cells styled as chips (like grid headers).
- Both work in `days` AND `range` pick modes (they mutate `selectedDates` directly, never `rangeDraft`).

## 3. Grid views (`AvailabilityGrid.tsx`)

### 3.1 View state

`viewSize: 'week' | 'month' | 'all'`. Mobile options: week/month (default week). Desktop options: week/month/all (default all). One SegmentedControl replaces the current pageSize control; switching resets `pageIdx`. Derivations:
- `paged = viewSize !== 'all'` — pages/`visibleColumns` pagination logic (currently isMobile-gated) now keys on `paged`; `'all'` keeps today's every-event-date columns.
- `eventDaysOnly` toggle renders whenever `paged` (mobile AND desktop).
- Sticky time column / scrollable-month behavior: keyed on `paged && viewSize === 'month'` as today, viewport-independent.

### 3.2 Paint mode

- `showPaintToggle = isMobile` (both views). `effectivePaintMode = isMobile ? paintMode : true`. Default paintMode stays as the store default (on).
- Desktop: no toggle; painting always active.

### 3.3 Desktop wheel → horizontal scroll

Ref on the table's `overflow-x-auto` container. Effect (desktop only, i.e. `!isMobile`): non-passive `wheel` listener — when `el.scrollWidth > el.clientWidth` and `|deltaY| > |deltaX|`: `el.scrollLeft += e.deltaY; e.preventDefault()`. When the table fits, the listener does nothing (page scrolls normally). Cleanup on unmount/mobile switch.

## 4. Error handling / edge cases

- `toggleDays` dedupes by `toDateString`; empty candidate sets no-op (e.g., weekday entirely in the past).
- Week-number clicks spanning out-of-month days select only the displayed month's days (v9 passes the week's dates; filter to visible months and `>= today`).
- View switch clamps `pageIdx` (existing render-phase clamp covers it).
- Wheel handler must not capture when unscrollable, and must not attach on mobile (touch scrolling untouched).

## 5. Testing

- Unit: `toggleDays` (add-missing, remove-all-present, past-filter interplay tested via pre-filtered candidates, dedupe) added to `dateRange.test.ts`.
- Probes: separate selected-dates card exists below calendar; weekday-header click selects/deselects that column (count changes symmetric); week-number click selects the row; desktop All|Week|Month control pages correctly and shows the event-days toggle in paged views; mobile week view shows the paint toggle; synthesized WheelEvent on an overflowing desktop grid moves `scrollLeft` and leaves it unchanged when not overflowing.
