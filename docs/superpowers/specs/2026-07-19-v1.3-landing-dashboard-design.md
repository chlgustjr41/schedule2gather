# schedule2gather v1.3 Landing + Dashboard — Design Spec

**Date:** 2026-07-19
**Status:** Approved by user (brainstorm; landing structure and dashboard controls chosen via questions)
**Lands on:** branch `redesign` → new PR to main
**Parent specs:** v1.0 redesign, v1.1 follow-up, v1.2 mobile UX

## 1. Scope

1. **Landing page** at `/`: hero, join-by-code entry with inline validation, Create CTA.
2. **Create page** at `/new`: the existing `CreateEventForm` moves there unchanged.
3. **Dashboard** at `/dashboard`: Google-sign-in-gated list of the user's created events with open / copy link / reopen / finalize-deep-link / delete, plus joined & painted counts and finalized badges.
4. **`deleteEvent` Cloud Function**: owner-checked recursive delete (the only backend addition).
5. **Shared `AppHeader`** with auth-aware nav, replacing the ad-hoc headers on Landing/Create/Event pages.

Out of scope: composite indexes (client-side sort instead), inline finalize from the dashboard (deep-links to the event page), account management beyond the existing Google sign-in, email/password auth.

## 2. Routes & Pages

| Route | Page | Content |
|---|---|---|
| `/` | `LandingPage` (rewritten) | `AppHeader` · hero (headline + subline) · JoinByCode card · primary CTA → `/new` |
| `/new` | `CreatePage` (new, thin) | `AppHeader` · the existing `CreateEventForm` (moved verbatim) |
| `/dashboard` | `DashboardPage` (new) | `AppHeader` · signed-out state or event list |
| `/e/:slug` | `EventPage` | unchanged except its inline header is replaced by `AppHeader` |

### 2.1 `AppHeader` (new, `src/components/AppHeader.tsx`)

`Wordmark` left; right side: **Dashboard** link (visible when `user && !user.isAnonymous`), **Sign in** ghost button (visible when anonymous/no user; calls `authStore.signInWithGoogle`, which links the anonymous uid — events created before signing in keep their owner), and `ThemeToggle`. Max-width matches page content (`max-w-5xl` on event/dashboard, `max-w-2xl` on landing/create — accept `className` passthrough).

### 2.2 JoinByCode card (part of LandingPage)

- `TextField` "Event code" (placeholder "e.g. 6w3wrh") + "Join" `Button`.
- Normalization helper `normalizeCode(raw): string` in `src/lib/joinCode.ts`: trim, lowercase, strip anything matching a full share URL (accepts a pasted `…/e/abc123` and extracts the slug), allow `[a-z0-9-]`.
- On submit: `getEvent(code)` (new one-shot `getDoc` in eventService). Exists → `navigate('/e/' + code)`. Missing → inline "No event found with that code — check for typos." Empty input → button disabled. Lookup failure (offline) → inline "Couldn't check the code — are you online?"

### 2.3 LandingPage content

Hero: headline "Find a time to meet, fast.", subline "Pick dates, share a link, see when everyone's free. No sign-ups needed to vote." Primary CTA `Create an event` → `/new` (Button lg). JoinByCode card below the CTA. Nothing else (YAGNI).

### 2.4 DashboardPage

- **Gate:** if `!user || user.isAnonymous` → explainer card ("See and manage every event you've created") + "Sign in with Google" button (existing `signInWithGoogle`); on success the list loads.
- **Data:** `listMyEvents(uid)` (new in eventService): one-shot `getDocs(query(collection(db,'events'), where('ownerUid','==',uid)))`, client-sorted by `createdAt` desc (no composite index needed). Per event, two aggregate **count** queries (`getCountFromServer`): joined = participants collection; painted = participants where `availability != ''`. Counts load after the list (per-row placeholders); if the painted count query rejects, hide that figure only.
- **Row:** event name · date range ("Jul 27 – 29" from first/last of `dates`; recurring mode shows "weekly") · joined/painted ("4 joined · 3 voted") · `✅ Finalized` badge when set · created date (muted). Actions: row click → `/e/{slug}`; copy-link icon button (clipboard, "Copied ✓" flash); when finalized a **Reopen** button (existing `reopenEvent`, optimistic row update); when not finalized a **Finalize…** link → `/e/{slug}`; **Delete** (danger ghost) → confirmation `BottomSheet` ("Delete '<name>'? All votes and comments are removed permanently.", danger confirm) → `deleteEvent` callable → remove row on success; inline row error on failure.
- **States:** loading skeleton rows; empty ("Nothing yet — create your first event" + CTA to `/new`); query error with Retry button.
- After sign-OUT (header still allows via existing flows elsewhere; dashboard shows the signed-out gate again).

## 3. Backend: `deleteEvent` callable

`functions/src/deleteEvent.ts` (+ export in `functions/src/index.ts`):

- Input `{ slug: string }`; validated by pure helper `assertDeletable(data: { ownerUid?: string } | undefined, uid: string | undefined): 'ok' | 'unauthenticated' | 'not-found' | 'permission-denied'` in `functions/src/lib/deleteGuard.ts` (unit-tested).
- Flow: reject unauthenticated; `get(events/{slug})`; reject missing / non-owner; `firestore.recursiveDelete(eventRef)`; best-effort `admin.database().ref('presence/'+slug).remove()` in try/catch (RTDB may be unprovisioned); return `{ ok: true }`.
- Client: `deleteEventRemote(slug)` in eventService via `httpsCallable(functions, 'deleteEvent')` (reuses the existing functions client from `createEvent`).
- CI already deploys functions on merge. No rules changes (list/read/count already public; delete goes through the function).

## 4. Error handling summary

Join code: inline unknown-code / offline messages, disabled empty submit. Dashboard: gate, skeletons, empty state, query retry, per-row action errors inline, painted-count graceful hide. Delete: confirmation required, in-flight disabled state, server errors surfaced ("Couldn't delete — try again"). deleteEvent function: typed HttpsError codes (`unauthenticated` / `not-found` / `permission-denied` / `invalid-argument` for bad slug shape).
